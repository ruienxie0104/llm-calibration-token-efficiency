Four Core Gaps

Confirmed by 16 systematic arXiv searches
Gap 1 ★★★

Self-assessment accuracy → Token prediction

NOBODY has studied whether LLM self-assessment quality can predict token requirements.

Why it matters: If it works, we have a training-free token allocation signal — the model "knows itself."

Gap 4 ★★★

Calibration → Token savings causal chain

Nobody connects calibration improvement to token efficiency gains.

學姐 proved IDS improves calibration. Others proved difficulty-aware allocation reduces tokens. Nobody linked them.

Gap 2 ★★

IRT difficulty signals × Token allocation

Nobody uses IRT objective difficulty to guide token distribution. IRT provides difficulty relative to model ability — more principled than ad-hoc estimators.

Gap 3 ★★

Verbalized confidence as token signal

UNTESTED. Kumaran (2026-06) found verbalized conf tracks commitment, not correctness. Risk, but also a research question — which signal works for token allocation?

One-sentence story

學姐 proved IRT difficulty signals improve LLM self-assessment (LCAE) and mentioned reliability-inference cost association. This research fills that gap: we measure the self-assessment — token efficiency relationship, validate the calibration → token savings causal chain, and propose a self-assessment-based token allocation strategy. If it holds, this provides a training-free token optimization method — no extra training needed, just let the model know itself better.

Phase 1: LCAE × Token Correlation

Does self-assessment accuracy predict token efficiency?

Measured Variables

VariableSourceDescription
LCAEfrom 學姐Self-assessment vs IRT consistency
Ability (θ)from 學姐IRT estimated model capability
Difficulty (b)from 學姐IRT estimated item difficulty
Token usage (T)NEWActual reasoning tokens per question
Accuracyfrom 學姐Answer correctness
Token efficiencyNEWAccuracy / Token usage

Steps

1. Run 20 models on 學姐 benchmark, measure reasoning tokens per question

2. Compute LCAE vs avg token efficiency correlation (Pearson/Spearman)

3. Stratified analysis by difficulty: easy / medium / hard

4. Analyze: do calibrated models show adaptive allocation (easy=less, hard=more)?

Expected: H1 holds → LCAE positively correlates with token efficiency, especially on hard questions. H1 null → self-assessment and token efficiency are unrelated (also an important finding).

Risk Assessment

Risks and mitigation strategies
High
Verbalized confidence may not work for token allocation
Kumaran (2026-06): tracks commitment, not correctness
Mitigation: Test both verbalized and log-prob; frame as research question; IDS may still work independently
High
Causal chain may not hold
Better calibration might not cause token savings
Mitigation: Null result is itself a finding; distinguish allocation quality from total volume; report honestly
Med
Scope may be too narrow
Only one intersection: self-assessment × token
Mitigation: Broaden to metacognition × compute allocation; 4 contributions cover multiple levels
Med
學姐 data may not have token records
Need to re-run experiments
Mitigation: Confirm data format first; if needed, run representative subset (5-8 models)
Low
ROI-Reasoning too close
